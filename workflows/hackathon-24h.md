# 24-Hour Hackathon: The Shortest Path

Timeline for a solo or small team delivery using GHOS.

## Phase 1: The Blueprint (Hour 0 - 2)
- **Task 1:** Initialize repo + GHOS.
- **Task 2:** Invoke `agents/build-council.md` to define the "Demo Path".
- **Task 3:** Scaffold the project structure. **Verify build immediately.**

## Phase 2: Core Execution (Hour 2 - 16)
- **Rule:** 1 Feature = 1 Plan -> N Implementation Turns -> 1 Verification.
- **Priority:** User-facing UI and core API logic.
- **Review:** Use `workflows/diff-first-review.md` for every major merge.

## Phase 3: The Pivot/Prune (Hour 16 - 18)
- **Check:** Are we 80% done with the Demo Path?
- **Action:** If not, use `prompts/prune-scope.md` to cut non-essential features.

## Phase 4: Polish & Demo Prep (Hour 18 - 22)
- **Task 1:** CSS/UI cleanup for "visual impact".
- **Task 2:** Run `agents/ship.md` to generate README and Demo Script.
- **Task 3:** Record a fallback demo video.

## Phase 5: Submission (Hour 22 - 24)
- **Task 1:** Final `workflows/final-ship-checklist.md`.
- **Task 2:** Submission text + pitch deck finalization.
- **Task 3:** Submit 30 mins early.
